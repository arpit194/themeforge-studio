# ThemeForge Studio

A React + Vite app for visually displaying and testing ThemeForge design tokens.

## Project structure

```
src/
  routes/               # TanStack Router file-based routes — minimal code only
  components/
    home/               # Components for the / route
    shared/             # Reusable components shared across multiple routes
    ui/                 # Styled headless components (Button, Tabs, etc.)
  index.css             # Global reset styles
  main.tsx              # App entry: RouterProvider wrapped in ThemeProvider
  routeTree.gen.ts      # Auto-generated by TanStack Router plugin — do not edit
```

## Conventions

- **Routes are thin** — route files only declare `Route` with `createFileRoute` and point to a component. No logic or JSX in route files.
- **Each route has its own folder in `src/components/`** named after the route (e.g. `home/`, `about/`). The main page component plus any route-specific sub-components live inside that folder.
- **Shared components go in `src/components/shared/`** — only move a component here when it is used by more than one route.
- No `App.tsx` — the root component concept is replaced by `src/routes/__root.tsx`.

## Routing

- File-based routing via `@tanstack/router-plugin` (`tanstackRouter` Vite plugin).
- Routes auto-discovered from `src/routes/`.
- `__root.tsx` — root layout with `<Outlet />`.
- `index.tsx` — renders at `/`.

## ThemeForge

- `@arpit194/themeforge` is installed as a dependency.
- `<ThemeProvider>` wraps the entire app in `main.tsx`.
- CSS vars follow the pattern: `--{cssVarPrefix}-{category}-{key}` e.g. `--tf-color-primary-500`.
- Token categories: `color`, `spacing`, `radius`, `shadow`, `font-family`, `font-size`, `font-weight`, `line-height`, `letter-spacing`, `semantic`.
- Custom tokens are added via module augmentation in `main.tsx` (e.g. `CustomColorKeys`, `CustomRadiusKeys`).

## Color usage rules

- **Always use semantic color tokens for all color values in components** — never use primitive color vars (e.g. `--tf-color-primary-500`) directly.
- Semantic vars follow the pattern `--tf-color-{semantic-key}` e.g. `--tf-color-bg-primary`, `--tf-color-text-on-primary`, `--tf-color-border-default`.
- Semantic tokens are grouped by: `bg`, `text`, `icon`, `border` — each with role + state variants.
- States available: `hover`, `active`, `disabled`, `selected`, `subtle`, `focus`.
- Status colors (`success`, `warning`, `error`, `info`) are available in all groups e.g. `bg-error`, `text-error`, `border-error`.
- Primitive color vars are only acceptable inside ThemeForge itself when defining the semantic layer defaults.

## UI Components

- Base UI (`@base-ui/react`) is the headless component library used for accessibility primitives.
- Styled Base UI components live in `src/components/ui/` — one file per component (e.g. `Button.tsx`, `Tabs.tsx`).
- All styles are applied using ThemeForge semantic CSS variables — no hardcoded colors, no primitive color vars.
- Component variants are managed with `class-variance-authority` (CVA). Each component has a paired CSS Module (e.g. `Button.tsx` + `Button.module.css`). CVA maps variant names to CSS Module class names.

## Scripts
